【AI協業運用ルール】DesignFactory リポジトリ

作成日：2026年5月11日
対象：Claude Code / Codex / 将来参加するすべてのAIアシスタント

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. リポジトリに入れてよいもの
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- HTML / CSS / JavaScript などのサイトファイル
- 画像・アイコン等のアセット
- _works/ 配下の作業ログ・運用文書（機密情報を除く）
- .gitignore
- README.md（作成する場合）


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2. リポジトリに入れてはいけないもの
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- .claude/ 配下のすべてのファイル（Claude Code のローカル設定）
- .codex/ など特定AIツール専用の設定フォルダ
- .env / .env.local / *.local.json
- _works/GitHub_Token.txt
- その他APIキー・Personal Access Token・SSH鍵・認証情報を含むファイル
- 他のAIツールの作業権限を制限する設定ファイル

※ 誤ってコミットした場合は、削除だけでなくトークンの即時失効・再発行を行うこと


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3. GitHub同期に使うアカウント
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- GitHubリポジトリ：inuusagineko-design/DesignFactory
- Cloudflare Pages連携元：inuusagineko-design/DesignFactory
- Linuxサーバーからのpush認証：inuusagineko-design に統一
- koita5959-ux は使用しない（認証キャッシュが残っている場合は整理すること）


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. Cloudflare Pagesへの反映手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Cloudflare Pages は GitHub の master ブランチを参照して自動デプロイされる。
以下の順序で確認すること。

  1. ローカルで変更をコミット
  2. GitHub の master へ push
  3. Cloudflare Pages の管理画面でデプロイ完了を確認
  4. https://dfactory.pages.dev/ で最新表示になっていることを確認

※ Linuxサーバー上の作業だけでは公開環境は更新されない


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
5. 各AIの役割と作業ルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- 同じファイルへの並行編集は避ける
- 作業前に、何を触るかをコミットメッセージまたは _works/ 内のログに記録する
- master へ push した後は、変更内容を西村様へ報告する
- 問題が発生した場合は即座に作業を停止し、西村様へ報告する
- 謝罪より再現防止を優先し、何を変更したか・何が残るリスクかを明確に伝える


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
6. 本番反映前の確認手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  □ 機密ファイルがステージに含まれていないこと（git status で確認）
  □ .claude/ 等のAI設定ファイルがステージに含まれていないこと
  □ コミットメッセージが変更内容を正確に表していること
  □ push 先が inuusagineko-design/DesignFactory であること
  □ Cloudflare Pages のデプロイが正常に完了していること
  □ 公開URL（https://dfactory.pages.dev/）で表示を確認していること


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
7. さくらサーバー側との役割分担
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

現時点では、公開環境はCloudflare Pages（GitHubのmasterブランチ参照）を主軸とする。
さくらサーバーとの役割分担については、西村様の指示に従い別途定める。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

このルールは、Claude Code・Codex を問わずすべての協業者が遵守すること。
変更が必要な場合は西村様の承認を得てから更新すること。
